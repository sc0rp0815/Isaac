CORE IDENTITY OF ISAAC
----------------------
Isaac is intended to become a local, stateful, evolving AI system.

Long-term target:
1. receive input
2. classify interaction correctly
3. retrieve relevant prior context BEFORE response strategy
4. choose explicit response strategy
5. execute deterministically
6. evaluate outcomes
7. store structured memory
8. learn gradually from interactions and corrections
9. evolve without losing clarity, stability, or identity boundaries

Isaac is NOT meant to remain:
- a stateless chatbot
- a generic prompt wrapper
- a tool shell around an LLM
- an unstructured agent loop

Isaac should become a cognitive kernel.

ARCHITECTURAL PRINCIPLES
------------------------
These principles are binding:
1. Classification must control routing.
2. Retrieval must happen before response strategy selection.
3. Executor must execute, not reinterpret decisions.
4. Memory must be typed and structured.
5. Lightweight social inputs must short-circuit locally.
6. Normal chat must not opportunistically trigger tools.
7. Strategy must be explicit and inspectable.
8. Persistence ownership must be clear.
9. Inquiry/clarification must eventually be deterministic and justified.
10. Learning must be gradual, auditable, and bounded.
11. Trust is postponed for now; owner interactions can be assumed high-trust by default.
12. Architecture must remain incremental and debuggable.

EVOLUTION 2.0 CONTEXT
---------------------
Evolution in Isaac does NOT mean:
- uncontrolled self-rewriting
- random personality drift
- free mutation of core identity
- broad autonomous architecture change

Evolution in Isaac means:
- persistent memory is used meaningfully
- repeated feedback changes behavior gradually
- preferences stabilize over time
- inquiry becomes less necessary as context becomes richer
- learned corrections reduce future mistakes
- changes remain inspectable and attributable
